# 全局控制面板功能 - 优化说明

## 🎯 实现目标

为火星臭氧可视化平台添加交互式控制面板，让用户可以选择：
- **火星年 (MY)**: 27, 28
- **季节 (Ls)**: 0-360° 滑动条

所有可视化视图（3D地球、热力图、折线图）都会实时响应用户的选择。

---

## 📂 新增文件

### 前端
1. **`frontend/src/contexts/DataContext.jsx`**
   - React Context，全局状态管理
   - 存储：`marsYear`, `solarLongitude`
   - 提供 Hook: `useDataContext()`

2. **`frontend/src/components/ControlPanel.jsx`**
   - 控制面板 UI 组件
   - 火星年下拉菜单
   - 季节滑动条（0-360°，显示对应季节名称）

---

## 🔧 修改文件

### 前端

#### `frontend/src/App.jsx`
- 用 `<DataProvider>` 包裹整个应用
- 在左侧控制栏添加 `<ControlPanel />`

#### `frontend/src/services/api.js`
- 所有 API 函数增加参数支持：
  ```javascript
  fetchMapData(marsYear, solarLongitude)
  fetchSeasonalChart(marsYear)
  fetchSeasonalBands(marsYear)
  ```

#### `frontend/src/components/Mars3DViewer.jsx`
- 从 Context 获取参数：`const { marsYear, solarLongitude } = useDataContext()`
- useEffect 依赖项添加 `[marsYear, solarLongitude]`
- 参数变化时自动重新请求数据

#### `frontend/src/components/SeasonalChart.jsx`
- 从 Context 获取 `marsYear`
- 图表标题显示当前火星年：`MY${marsYear} - ...`

#### `frontend/src/components/SeasonalLineChart.jsx`
- 从 Context 获取 `marsYear`
- 图表标题显示当前火星年

---

### 后端

#### `backend/services/data_service.py`
增强功能：

1. **`_find_file_by_ls(mars_year, solar_longitude)`**
   - 根据火星年和季节自动查找匹配的 NC 文件
   - 解析文件名中的 Ls 范围（如 `ls2_my27_ls17.nc`）
   - 返回最匹配的文件

2. **`get_point_cloud_data(mars_year, solar_longitude)`**
   - 参数化查询（原来固定使用一个文件）
   - 自动查找最接近目标 Ls 的时间点

3. **`get_zonal_mean_data(mars_year)`**
   - 支持指定火星年
   - 自动聚合该年的所有 NC 文件

4. **`get_latitudinal_bands_data(mars_year)`**
   - 支持指定火星年
   - 只处理该年的数据

#### `backend/main.py`
路由增加查询参数：


## 🔄 数据流

```
用户操作
  ↓
ControlPanel 更新 Context 状态
  ↓
组件监听 Context 变化（useEffect 依赖）
  ↓
调用 API（带参数：my=27, ls=100）
  ↓
后端接收参数 → 服务层查找对应文件
  ↓
返回数据 → 前端更新可视化
```

## 🎨 用户体验

### 控制面板功能
- **火星年选择器**：下拉菜单，选择 MY27 或 MY28
- **季节滑动条**：拖动查看不同季节（0-360°）
- **实时反馈**：
  - 显示当前 Ls 值和对应季节名称
  - 春季(0-90°) | 夏季(90-180°) | 秋季(180-270°) | 冬季(270-360°)

### 视图响应
- **3D地球**：显示"正在加载 MY27 Ls=100° 数据..."
- **热力图**：标题更新为 "MY27 - Ls-Latitude..."
- **折线图**：标题更新为 "MY27 - Seasonal Ozone..."
