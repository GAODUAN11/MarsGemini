# 火星臭氧预测与可视化系统 (MarsGemini) 项目分析报告

本报告根据当前代码库 `d:\A-development-project\MarsGemini` 的实际结构生成，旨在帮助初学者快速理解项目的核心功能、整体架构以及使用的现代 Web 开发技术栈。

## 一、 项目整体架构

本项目采用标志性的 **前后端分离架构** (Client-Server Architecture)：

- **前端 (Frontend)**：作为客户端，运行在用户的浏览器中，负责提供绚丽的用户界面(UI)、响应用户交互，并向后端发送数据请求。
- **后端 (Backend)**：作为服务端，运行在服务器上，负责直接与本地的科学数据文件交互，执行复杂的时空数据切片和统计计算，最后以轻量级格式返回给前端。

这种架构的优势在于：前端只负责“貌美如花”（展示交互），后端只负责“赚钱养家”（数据处理）。两者通过标准化的 API 接口进行通讯。

## 二、 核心功能模块与技术实现解析

### 模块一：后端数据处理引擎

后厨的指挥中心，将晦涩的科学气象数据（OpenMARS 数据库的 `.nc` 文件）转化为前端可视化的燃料。

*   **功能 1：数据检索与路由**
    *   **实现逻辑**：接收前端对特定年份 (MY) 和季节 (Ls) 的查询请求。
    *   **核心技术**：`FastAPI` (Python)。一个现代、快速的高性能 Web 框架。
*   **功能 2：点云空间切片提取**
    *   **实现逻辑**：从气象文件中提取指定时间截面上的全球臭氧柱浓度，剔除无效值，并转换为离散的点云坐标列表。
    *   **核心技术**：`xarray` + `numpy` (Python)。用于处理多维科学数据（特别是带有经纬长时序维度的 NetCDF 数据）。
*   **功能 3：时空维度聚合计算**
    *   **实现逻辑**：对原始气象数据进行降维操作，如计算全局纬向平均（沿所有经度求该纬度圈的均值），或者计算特定纬度带的区域均值，生成时序变化数据。

### 模块二：前端核心交互与状态管理

所有视图组件的“大脑”，确保数据在页面上的所有角落同步更新。

*   **功能：全局状态与组件通信**
    *   **实现逻辑**：创建一个全局的数据上下文，统一管理用户当前选择的参数（年份、季节）。当用户在控制面板修改这些参数时，所有的图标和 3D 地球都能自动侦测变化并请求新数据。
    *   **核心技术**：`React Context API` + `React Hooks (useState, useEffect)` (JavaScript)。

### 模块三：3D 地球与手势交互视图 (GlobePage)

项目中最具视觉冲击力的模块，将火星数据映射到 3D 球体上。

*   **功能 1：3D 点云渲染**
    *   **实现逻辑**：在网页中渲染一个可旋转、缩放的逼真火星表面，并将后端的点云数据根据数值映射为不同的颜色和高度，投射到对应的经纬度坐标上。
    *   **核心技术**：`react-globe.gl` (JavaScript, 基于 Three.js 封装)。
    *   **色彩引擎**：`d3-scale` & `d3-scale-chromatic` (用于将臭氧浓度值转化为彩虹色域 Viridis)。
*   **功能 2：AI 手势控制**
    *   **实现逻辑**：调用用户电脑摄像头捕捉手部动作，识别特定的手势（如捏合拖拽），从而不使用鼠标就能旋转 3D 地球。
    *   **核心技术**：`@mediapipe/hands` (Google 的机器视觉库) + `react-webcam`。

### 模块四：多维数据统计看板 (DashboardPage)

提供专业的科学数据平面制图功能，便于进行趋势分析。

*   **功能 1：季节-纬度热力图**
    *   **实现逻辑**：以横轴为时间（太阳黄经 Ls），纵轴为纬度，用颜色深浅（Heatmap）展现臭氧浓度的时空演化规律。
    *   **核心技术**：`react-plotly.js` (基于 Plotly.js 的图表库，非常适合科学绘图)。
*   **功能 2：纬度带变化折线图**
    *   **实现逻辑**：将火星划分为极地、中纬度、赤道等五个区域，绘制出各区域臭氧均值随季节变化的对比曲线。
    *   **核心技术**：同上 (`react-plotly.js`)。

## 三、 技术栈清单及依赖关系汇总

### 后端 (Python 生态)
*   **Web 框架**: `FastAPI` (提供高性能的 API 接口)
*   **多维数据处理**: `xarray` (处理 .nc 格式的科学气象数据)
*   **科学计算核心**: `numpy` (极速矩阵运算与数组构建)
*   **服务器**: `uvicorn` (ASGI 服务器，用于运行 FastAPI)

### 前端 (JavaScript/React 生态)
*   **视图框架**: `React v19` (利用虚拟 DOM 构建用户界面的行业标准)
*   **构建工具**: `Vite` (极速的本地开发服务器和打包工具，替代了老旧的 Webpack)
*   **网络通信**: `axios` (基于 Promise 的 HTTP 客户端，用于调用后端接口)
*   **3D 全局渲染**: `react-globe.gl` (在网页端运行的 WebGL 3D 引擎)
*   **2D 图表渲染**: `react-plotly.js` (专业的科学级图表绘制)
*   **机器视觉/AI**: `@mediapipe/hands` + `@mediapipe/camera_utils` (赋能计算机视觉，实现手势追踪)
